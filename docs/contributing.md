# Contributing to Wrapster Project Template

We welcome contributions to the Wrapster Project Template! By contributing, you're helping us create a better tool for everyone.

Please take a moment to review this document to make the contribution process as smooth as possible.

## Code of Conduct

We adhere to a [Code of Conduct](code_of_conduct.md) to ensure a welcoming and inclusive environment. Please read it before participating.

## How Can I Contribute?

### Reporting Bugs

*   **Ensure the bug hasn't already been reported.** Search through existing issues to see if someone has already reported it.
*   **Provide a clear and concise description of the issue.** Explain the problem and how to reproduce it.
*   **Include relevant details.** This could be your operating system, Python version, or any error messages.

### Suggesting Enhancements

*   **Check existing suggestions.** See if your idea has already been proposed.
*   **Describe your idea clearly.** Explain what the enhancement is and why it would be useful.
*   **Provide examples if possible.** How would this enhancement work in practice?

### Code Contributions

*   **Fork the repository** and create your branch from `main`.
*   **Set up your development environment.** Refer to the `README.md` for instructions.
*   **Follow existing coding style and conventions.** We use `ruff` for linting and `black` for formatting.
*   **Write clear, concise commit messages.** Explain the *what* and *why* of your changes.
*   **Add tests** for new features or bug fixes to ensure they work correctly and prevent regressions.
*   **Ensure all tests pass** before submitting your pull request.

### Pull Request Process

*   **Open a pull request** with a clear title and description.
*   **Reference any related issues** in your pull request description (e.g., "Fixes #123").
*   **Be responsive to feedback.** We'll review your changes and may suggest improvements.
*   **Keep your pull request focused.** Address one issue or add one feature per pull request.

## Development Setup

Refer to the main `README.md` file in the project root for instructions on setting up your development environment.

Thank you for contributing!
